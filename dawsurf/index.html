dawsurf - tap here to launch!

A VST parameter touch controller for Reaper (experimental)

If you’re lucky enough to have a smartphone, tablet or touch screen monitor and your Reaper project has lots of VSTs then this might interest you:

Features

  • runs on a smartphone or tablet running Android Chrome, iPad/iPhone Chrome, iPad/iPhone Safari, or Opera Mobile
  • runs on a desktop pc with touchscreen monitor, running latest Chrome (recommended) or Firefox
  • navigate using pinch zoom and finger panning
  • tap a track, VST or group to toggle expand or collapse
  • 3 ways to change a parameter value: (a) tap then drag, (b) direct drag, or (c) tap to toggle between min/max (off/on)
  • dynamic layout attempts to maximise the number of parameters visible
  • auto grouping lets you manage VSTs with thousands of parameters
  • optional independent vertical scrolling of tracks (desktop only)
  • themeable via sass/css. Currently there are 2 themes: pastel and retro-pet
  • runs over TCP/IP websocket (no midi), even over the internet!
  • ability to download and run the app offline
  • connect multiple touch devices to a single Reaper instance

Limitations

  • doesn’t control other things like volume, mute/solo, sends, transport, etc
  • some VSTs behave badly and expose empty or duplicate parameter names or don’t expose all parameters
  • only supports Reaper running on Windows not Mac
  • doesn’t work in Microsoft Internet Explorer

Usage

Currently this app is in beta test i.e. there may be bugs (disclaimer) so please lower your expectations and don’t install the server components on a mission critical machine!

You’ll need to be running Reaper v4.20+ on Windows and your app browser has to be Firefox 10+ or Chrome 16+. And you need a smartphone or tablet running Android 4.0+ or iOS, or a desktop pc with a multi-touch monitor (e.g. iiyama Prolite T2250MTS).

Some components need to be installed on your Reaper machine but don’t worry, they can easily be uninstalled.

1. Install server components

On your Reaper box run the appropriate dawsurf installer: either Dawsurf.Setup.x86 or Dawsurf.Setup.x64 (1.3MB each). If asked, allow prerequisites to be downloaded from Microsoft and installed (around 55MB).

2. Configure Reaper

Start Reaper and select Preferences -> Control Surfaces -> Add -> Generic .NET wrapper. Leave all fields empty and press OK. The Reaper.dawsurf console window should immediately appear with some welcome text like this… 

	Reaper.dawsurf v2012.11.5.0
	Reaper v4.22/x64
	Creating WebSocketServer...done!
	Initialising WebSocketServer...done!
	Starting WebSocketServer...done!
	IP 192.168.0.100
	Server SCOOBY listening on port 8080...

3. Launch the app

Launch the app in your browser. Click Add… then fill in the dialog form fields:

  • Unique Id: anything you want e.g. Scooby-Doo
  • Server: the Reaper server name or IP address. Given the example above we’d enter SCOOBY or 192.168.0.100
  • Port: enter 8080 (the default)
  • Auto Connect: optionally tick to immediately connect when the app is launched

Now click Connect. If it fails click here for help.

4. Use it!

After a short pause you should see Master track appear along with the other tracks in a collapsed state, at which point:

  • tap a track, VST, or group to toggle expand/collapse
  • drag a parameter left/right to decrease/increase its value
  • tap a parameter to (a) show the slider if the value is between min/max or (b) toggle between min/max (off/on)
  • tap the background to toggle the exit button
  • pinch zoom in/out
  • pan left/right/up/down
  • drag a track up/down to scroll independently of others (desktop only with independent track scrolling enabled)

Some shortcut keys are available:

  • F11 to toggle full screen
  • ESC to exit
  • SPACE to toggle the exit button

FAQ

The connect fails, help !

  • check you can ping the Reaper box from the browser box
  • ensure port 8080 isn’t being blocked by a firewall. If it is, either unblock 8080 or use a different unblocked port.
  • check Reaper is running and the dawsurf console is showing ‘Server something-or-other listening on port something-or-other…’
  • check what you entered in the server config dialog is correct

It’s behaving oddly in Firefox !

Certain options need to be set as follows:

	Value	Setting
	-----	-------
	On	Options -> Content -> Javascript -> Allow scripts to disable or replace context menus
	On	Options -> Content -> Colours -> Allow pages to choose their own colors
	On	Options -> Privacy -> History -> Remember History
	Off	View -> Zoom -> Zoom Text Only

I highly recommend you disable all addons since they can cause problems, for example Grab and drag breaks scrolling and Vimperator aggressively swallows all keystokes.

How to I change the listening port from 8080 ?

In the Reaper.dawsurf console window, select Options -> Server and change the value. And don’t forget to update the port in the browser app.

Exactly what gets installed on my Reaper box ?

The following prerequisites are required:

  • Microsoft .NET Framework 4 Full (around 50MB)
  • Microsoft Visual C++ 2010 x86/x64 Redistributable SP1 (around 5MB)

If they’re not already installed, either manually download and install them from Microsoft, or let the dawsurf installer do it for you.

These files and folders are added to your Reaper folder:

	\Program Files
		\REAPER or \REAPER (x64)
			reaper.exe.config			<-- added
			\Plugins
				reaper_csurf_dotnet.dll		<-- added
				\Dizzib				<-- a few files added in here

Can I run the app offline from my local hard-disk ?

Yes but only if you’re using Chrome with the -allow-file-access-from-files switch. The app launches much faster locally over the file:/// protocol compared to http:// plus you no longer need to be connected to the internet!

Download and extract the required version – dev.zip 20121105.zip 20120529.zip – then open app/index.html.

How do I uninstall this crap off my Reaper box ?

Just re-run the installer and select Remove. The following prerequisites will need to be manually uninstalled via control panel:

  • Microsoft .NET Framework 4
  • Microsoft Visual C++ 2010 x86/x64 Redistributable

What’s with the silly name ?

  1. DAW + surfer = dawsurf
  2. damn + awesome + surface = dawsum + surface = dawsurf :)

Development

This project is experimental and open to further possibilities:

  • support Reaper sends, mute, solo, vol, pan, commands
  • support other DAWs with a capable API
  • make opensource
